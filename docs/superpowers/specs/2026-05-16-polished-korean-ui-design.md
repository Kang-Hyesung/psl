# Polished Korean UI Design Spec

## Goal

교보문고 상품 목록에 주입되는 `숨기기` 버튼, 확장 팝업, 옵션 페이지를 하나의 제품처럼 보이도록 정리한다. 핵심 목표는 `담기`와 `숨기기` 버튼의 크기/정렬 불균형을 없애고, 한국어 SaaS 관리 도구처럼 읽기 쉬운 시각 체계를 만드는 것이다.

## Scope

이번 작업은 UI와 표시 문구의 정돈에 집중한다. 숨김 목록 저장, 동기화, 삭제, 새로고침, 옵션 열기 같은 기존 기능 동작은 변경하지 않는다.

포함 범위:

- 교보 목록에 주입되는 `숨기기` 버튼 스타일 정리
- 팝업 버튼 계층과 상태 표시 정리
- 옵션 페이지 요약 카드, 테이블, 삭제 액션 정리
- 한국어 문구와 테스트 기대값 정합성 유지
- 전체 플로우 캡처 QA

제외 범위:

- 새 기능 추가
- 새 런타임 의존성 추가
- 교보문고 실제 DOM 구조를 대규모로 재해석하는 어댑터 변경
- 서버/API/저장소 구조 변경

## Current Problems

### Product List Injection

교보 목록에 삽입되는 `숨기기` 버튼은 현재 기존 `담기` 버튼과 시각적 체계가 맞지 않는다. 버튼 높이, 너비, 패딩, 색상, 정렬 기준이 다르게 보이면 사용자는 확장 UI를 임시로 붙인 요소처럼 느낀다.

또한 `숨기기`는 상품 카드에서 보조 액션이어야 한다. `담기`와 같은 무게의 주요 구매 액션처럼 보이면 흐름을 방해하고, 너무 작은 텍스트 버튼처럼 보이면 클릭 가능한 기능으로 인식하기 어렵다.

### Popup

팝업은 현재 SaaS 스타일의 기본 구조를 갖췄지만 버튼들이 모두 하단에 모여 있다. `숨김 끄기`, `상태 새로고침`, `숨김 목록 열기`, `옵션 열기`가 같은 영역에서 경쟁하면서 우선순위가 약하다.

### Options

옵션 페이지는 요약 카드와 테이블이 생겼지만, 삭제 계열 액션과 일반 액션의 시각적 체계가 더 분명해야 한다. 테이블 행의 작업 버튼도 팝업과 같은 버튼 체계를 공유해야 한다.

## Recommended Direction

추천 방향은 `B와 C의 중간안`이다.

- `숨기기`는 기존 교보 버튼과 크기/높이/정렬을 맞춘다.
- 색상은 강한 빨강이 아닌 연한 danger outline으로 둔다.
- `담기`와 경쟁하는 primary 버튼이 아니라, 관리 성격의 보조 danger action으로 보이게 한다.
- 팝업과 옵션의 삭제 계열 액션도 같은 danger tone을 사용한다.
- 전체 UI는 조용하고 밀도 있는 SaaS 관리 도구 톤을 유지한다.

## Design System

### Button Types

공통 버튼 체계는 네 가지로 둔다.

- `primary`: 가장 중요한 진행 액션. 예: `옵션 열기`
- `secondary`: 일반 보조 액션. 예: `상태 새로고침`, `숨김 목록 열기`, `새로고침`
- `danger`: 삭제/숨김처럼 되돌림이 필요한 관리 액션. 예: `숨기기`, `삭제`, `전체 삭제`
- `toggle`: 켜짐/꺼짐 상태를 바꾸는 액션. 예: `숨김 끄기`, `숨김 켜기`

버튼 높이는 기본 32px 이상으로 맞춘다. 같은 그룹 안의 버튼은 높이, border-radius, font-weight, padding 기준을 통일한다.

### Color

색상은 현재 `src/shared/saas-theme.css`의 중립 배경과 blue primary를 유지한다. danger 계열은 다음 원칙을 따른다.

- 기본 danger 텍스트: 어두운 red
- danger 배경: 아주 옅은 red
- danger border: 연한 red
- hover: 배경을 조금 더 진하게 하되 화면에서 과도하게 튀지 않게 한다.

### Typography

한국어 UI는 `system-ui`, `Segoe UI`, `Apple SD Gothic Neo`, `Noto Sans KR` 흐름을 유지한다. 버튼과 테이블 텍스트는 12~13px 범위에서 촘촘하지만 읽기 좋게 둔다.

## Product List UI

### Button Placement

`숨기기` 버튼은 기존 어댑터가 반환하는 `.prod_btn_area`에 삽입한다. 이 영역 안에서 기존 버튼과 같은 세로 흐름을 따른다.

### Button Style

주입 버튼은 다음 스타일을 가진다.

- `min-height: 32px`
- `width: 100%`
- `border-radius: 6px` 또는 교보 버튼과 어울리는 6~8px
- `font-size: 12px`
- `font-weight: 600`
- danger outline 색상
- hover/focus-visible 상태 제공

가능하면 클래스 하나로 스타일을 주입한다.

```css
.kyobo-hide-list-hide-button {
  width: 100%;
  min-height: 32px;
  border: 1px solid #f1b8b2;
  border-radius: 6px;
  background: #fff7f6;
  color: #b42318;
  font-size: 12px;
  font-weight: 600;
  cursor: pointer;
}
```

### Style Injection Strategy

content script가 교보 페이지에 버튼을 삽입하므로, 버튼 CSS도 content runtime에서 안전하게 주입해야 한다. 권장 방식은 `dom-runtime-cards.ts`에서 `style` 태그를 한 번만 추가하는 것이다.

요구사항:

- 중복 style 태그를 만들지 않는다.
- 스타일 태그에는 고유 marker attribute를 둔다.
- 호스트 페이지 CSS와 충돌하지 않도록 class selector 중심으로 작성한다.
- 교보 페이지가 없는 환경이나 테스트 DOM에서도 실패하지 않는다.

## Popup UI

### Layout

팝업은 현재 `교보 숨김 목록` 헤더와 상태 카드 구조를 유지한다. 다만 버튼 그룹은 우선순위를 더 명확히 한다.

권장 구조:

- 상단: 제목, 설명, 준비 상태 배지
- 중단: 숨긴 항목 수, 현재 탭, 숨김 기능 상태
- 상태 가까이: `숨김 끄기/켜기`
- 하단: `상태 새로고침`, `숨김 목록 열기`, `옵션 열기`

### Button Hierarchy

- `옵션 열기`: primary
- `상태 새로고침`: secondary
- `숨김 목록 열기`: secondary
- `숨김 끄기/켜기`: toggle 성격의 secondary 또는 warning-soft 버튼

버튼 텍스트는 모두 한 줄에 자연스럽게 들어가야 한다. 팝업 폭은 현재 360px 기준을 유지하되, 버튼 텍스트가 줄바꿈되지 않도록 grid column과 min-width를 조정한다.

## Options UI

### Summary Cards

요약 카드는 유지하되, 텍스트 위계를 더 선명하게 한다.

- `저장된 숨김 항목`: 숫자가 가장 중요한 정보
- `저장 위치`: 로컬
- `지원 사이트`: 교보문고

카드의 border, padding, heading 크기는 유지하되, 세 카드의 시각 밀도와 높이가 맞도록 한다.

### Table

테이블은 운영 도구답게 스캔하기 쉬워야 한다.

요구사항:

- 헤더 배경은 옅은 회색
- 행 높이는 충분하되 과하게 넓지 않게 유지
- `키`는 monospace로 유지
- `원본 링크`는 긴 URL이 행 밖으로 넘치지 않게 줄바꿈
- `삭제` 버튼은 danger outline이며 다른 버튼과 높이를 맞춘다.

### Empty State

빈 상태는 현재 문구 `저장된 숨김 항목이 없습니다.`를 유지하되, 테이블 아래에 작은 안내 박스처럼 보이게 한다. 필요하면 보조 문구를 추가할 수 있다.

권장 보조 문구:

```text
교보문고 상품 목록에서 숨기기를 누르면 여기에 표시됩니다.
```

## Testing

### Unit Tests

기존 unit tests는 유지한다. 문구 변경으로 실패하는 테스트가 있으면 한국어 기대값으로 업데이트한다.

### E2E Tests

E2E는 다음을 검증해야 한다.

- 팝업 헤딩 `교보 숨김 목록`이 보인다.
- 팝업의 탭 상태가 `교보문고 지원됨` 또는 `교보문고 아님`으로 표시된다.
- 팝업 토글 버튼이 `숨김 끄기`에서 `숨김 켜기`로 바뀐다.
- 옵션 헤딩 `숨김 목록 관리`가 보인다.
- 옵션 요약 카드 `저장된 숨김 항목`이 보인다.
- 옵션 테이블에서 `삭제` 버튼이 동작한다.
- `전체 삭제`, `새로고침` 버튼이 기존 기능을 유지한다.
- 교보 목록에서 `숨기기` 버튼이 삽입되고 클릭 시 카드가 숨겨진다.

### Visual QA

구현 후 기존 캡처 플로우를 다시 실행한다.

저장 경로 형식:

```text
qa-captures/full-flow-polished-ko-YYYYMMDD-HHMMSS/
```

필수 캡처:

- 교보 목록에 `담기`와 `숨기기`가 함께 보이는 화면
- 숨김 처리 후 화면
- 새로고침 후 숨김 유지 화면
- 동적 렌더링 후 숨김 재적용 화면
- 팝업 상태 화면
- 팝업 숨김 기능 off 화면
- 옵션 숨김 목록 화면
- 옵션 항목 삭제 후 화면
- 비지원 도메인 no-op 화면

## Acceptance Criteria

- `담기`와 `숨기기` 버튼이 같은 액션 그룹 안에서 어색하게 크기가 달라 보이지 않는다.
- `숨기기`, `삭제`, `전체 삭제`가 같은 danger action 계열로 인식된다.
- 팝업에서 주요 액션과 보조 액션의 우선순위가 더 명확하다.
- 옵션 페이지의 테이블과 요약 카드가 한국어 SaaS 관리 화면처럼 정돈되어 보인다.
- 모든 기존 기능 테스트가 통과한다.
- 새 캡처에서 한국어 문구가 깨지지 않고 표시된다.

## Risks

- 교보문고 실제 사이트 CSS와 주입 버튼 CSS가 충돌할 수 있다. 클래스 선택자와 명시적인 기본값으로 완화한다.
- `.prod_btn_area`의 실제 버튼 구조가 fixture와 다를 수 있다. 스타일은 버튼 자체에 직접 적용해 부모 구조 의존도를 낮춘다.
- danger 색상이 너무 강하면 교보 페이지 안에서 과하게 튈 수 있다. 기본 상태는 연한 outline으로 유지한다.
